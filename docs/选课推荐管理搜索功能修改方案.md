# 选课推荐管理面板 - 搜索功能修改方案

## 一、需求概述

在后台管理面板的「选课推荐」标签页中，增加两个搜索框：

1. **课程名称搜索框** - 按课程名称过滤推荐记录
2. **教师姓名搜索框** - 按教师姓名过滤推荐记录

两个搜索框可以同时使用，也可以单独使用，支持与现有的状态过滤器（全部/待审核/已公开）联合筛选。搜索为服务端过滤（后端 SQL LIKE 查询），搜索时自动重置到第一页。

---

| 文件路径                                    | 修改类型 | 说明                                        |
| ------------------------------------------- | -------- | ------------------------------------------- |
| `web/templates/admin/index.html`            | 修改     | 前端：添加搜索框 UI 及相关 JS 逻辑          |

| `services/course_recommendation/service.go` | 修改     | 服务层：GetAll 支持按课程名称和教师姓名过滤 |

## 三、后端修改方案

### 3.1 API 层 - `api/v1/course_recommendation/handler.go`

**修改函数：`GetAll`**（第 93-110 行）

当前 `GetAll` 函数从 query string 中读取 `page`、`page_size`、`status` 三个参数。需要新增两个可选参数：

| 参数名         | 类型   | 必填 | 说明                         |
| -------------- | ------ | ---- | ---------------------------- |
| `course_name`  | string | 否   | 课程名称搜索关键词，模糊匹配 |
| `teacher_name` | string | 否   | 教师姓名搜索关键词，模糊匹配 |

**修改内容：**

- 在现有的 `page`、`page_size`、`status` 参数获取代码之后，新增两行，使用 `c.Query("course_name")` 和 `c.Query("teacher_name")` 获取搜索参数（为空字符串时表示不过滤）。
- 修改 `services.GetAll()` 的调用，将 `courseName` 和 `teacherName` 作为新参数传入。

**修改后的函数签名调用方式：**

```
services.GetAll(page, pageSize, status, courseName, teacherName)
```

### 3.2 服务层 - `services/course_recommendation/service.go`

**修改函数：`GetAll`**（第 138-190 行）

当前函数签名：

```go
func GetAll(page, pageSize int, status string) ([]model.CourseRecommendation, int64, error)
```

修改后函数签名：

```go
func GetAll(page, pageSize int, status string, courseName string, teacherName string) ([]model.CourseRecommendation, int64, error)
```

**SQL 构建逻辑修改：**

当前的 WHERE 子句构建方式是简单的字符串拼接（仅处理 `status` 过滤）。需要改为使用条件数组 + 参数数组的方式，以支持多条件动态组合：

1. 创建 `conditions []string` 和 `args []interface{}` 两个切片
2. 根据 `status` 参数添加 `is_visible` 条件（与当前逻辑一致）：
   - `status == "pending"` → 添加条件 `is_visible = 0`
   - `status == "approved"` → 添加条件 `is_visible = 1`
   - `status == "all"` → 不添加 `is_visible` 条件
3. 若 `courseName` 非空，添加条件 `course_name LIKE ?`，参数为 `"%" + courseName + "%"`
4. 若 `teacherName` 非空，添加条件 `teacher_name LIKE ?`，参数为 `"%" + teacherName + "%"`
5. 将所有条件用 `AND` 连接，生成最终的 WHERE 子句
6. 在 COUNT 查询和分页查询中统一使用该 WHERE 子句和参数

**关键注意事项：**

- 分页查询中 `LIMIT ? OFFSET ?` 的参数需要追加到 `args` 切片末尾
- COUNT 查询使用的参数切片不包含 LIMIT/OFFSET 参数
- 当 `conditions` 为空时，不添加 WHERE 子句（即查询全部）

## 四、前端修改方案

### 4.1 Alpine.js 数据新增

**文件：`web/templates/admin/index.html`，第 602-830 行的 `adminPanel()` 函数**

在 `return { ... }` 对象中，`recommendationFilter` 属性（第 611 行）附近，新增两个搜索状态变量：

```
searchCourseName: ''
searchTeacherName: ''
```

### 4.2 搜索框 UI

**插入位置：** 在选课推荐标签页内容区域中，状态过滤器（第 328-345 行的 `<div class="bg-[#E5E5EA] p-1 rounded-xl flex">` 块）之后、推荐列表（第 349 行的 `<div class="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4">` 块）之前。

具体来说，在第 346 行 `</div>` 闭合标签（状态过滤器所在 `mb-6` 容器的结束）之后，第 348 行 `<!-- 推荐列表 -->` 注释之前，插入搜索栏 HTML。

**UI 设计要求：**

搜索栏使用与页面现有设计风格一致的样式（Apple 风格 UI，使用项目已有的 Tailwind CSS 类）：

- 整体容器：使用 `flex` 布局，包含两个搜索输入框，间距用 `gap-3`
- PC 端两个搜索框并排显示（`flex-row`），移动端竖向堆叠（`flex-col`）
- 每个搜索框使用与页面中其他输入框一致的样式类：
  - 容器：`bg-white rounded-xl shadow-sm`
  - 输入框：`bg-[#F2F2F7] border-none rounded-xl px-4 py-2.5 text-[15px] text-[#1C1C1E] placeholder-[#C7C7CC] focus:bg-white focus:ring-2 focus:ring-[#885021]/20 focus:outline-none transition-all duration-200`
- 搜索框左侧可选添加搜索图标（SVG 放大镜图标），与 placeholder 文字配合
- 课程名称搜索框 placeholder：`"搜索课程名称..."`
- 教师姓名搜索框 placeholder：`"搜索教师姓名..."`
- 搜索框使用 `x-model` 双向绑定到 `searchCourseName` / `searchTeacherName`
- 搜索框绑定 `@input` 事件（带防抖），触发搜索（也可以用 `@keyup.enter` 实现回车搜索，或两者兼用）
- 整个搜索栏容器底部间距 `mb-4`，与下方推荐列表保持间距

**防抖方案：**

在搜索框 `@input` 事件中使用 `setTimeout` + `clearTimeout` 实现防抖（延迟 300ms），避免用户每输入一个字符就发起请求。具体做法：

- 在 `adminPanel()` 的 data 中新增 `searchTimer: null` 属性
- 搜索框 `@input` 事件处理：清除旧 timer，设置新 timer，300ms 后调用搜索方法
- 或者更简单的方案：不做前端防抖，仅监听回车键 `@keyup.enter` 触发搜索，同时在搜索框旁边放一个"搜索"按钮

**推荐方案 - 回车搜索 + 搜索按钮：**

考虑到管理后台操作频率不高，采用更直观的方案：

- 两个搜索框绑定 `@keyup.enter` 事件触发搜索
- 搜索框右侧放一个"搜索"按钮，点击触发搜索
- 搜索按钮旁边放一个"重置"按钮，点击时清空两个搜索框并重新加载

### 4.3 搜索方法

**修改 `loadRecommendations()` 方法**（第 684-705 行）

在 axios GET 请求的 `params` 对象中，新增 `course_name` 和 `teacher_name` 参数：

```javascript
params: {
    page: this.pagination.page,
    page_size: this.pagination.pageSize,
    status: this.recommendationFilter,
    course_name: this.searchCourseName,
    teacher_name: this.searchTeacherName
}
```

后端会自动忽略空字符串参数（空字符串在 SQL 中会匹配 `%% `即所有记录）。

**新增 `searchRecommendations()` 方法：**

```javascript
searchRecommendations() {
    this.pagination.page = 1;
    this.loadRecommendations();
}
```

搜索时重置到第一页，然后重新加载数据。

**新增 `resetSearch()` 方法：**

```javascript
resetSearch() {
    this.searchCourseName = '';
    this.searchTeacherName = '';
    this.pagination.page = 1;
    this.loadRecommendations();
}
```

清空搜索条件并重新加载。

### 4.4 现有方法联动修改

**`changeFilter()` 方法**（第 707-711 行）：保持不变。切换状态过滤器时不清空搜索关键词，这样搜索条件和状态过滤可以联合使用。如果希望切换状态时清空搜索，也可以在 `changeFilter()` 中加入 `this.searchCourseName = ''; this.searchTeacherName = '';`，但从用户体验角度来说，保留搜索条件更合理。

## 六、交互逻辑总结

| 用户操作                             | 触发行为                                     |
| ------------------------------------ | -------------------------------------------- |
| 在搜索框中输入文字后按 Enter         | 重置页码为 1，带搜索参数重新请求后端         |
| 点击"搜索"按钮                       | 重置页码为 1，带搜索参数重新请求后端         |
| 点击"重置"按钮                       | 清空两个搜索框，重置页码为 1，重新请求后端   |
| 切换状态过滤器（全部/待审核/已公开） | 保留搜索关键词，重置页码为 1，重新请求后端   |
| 点击分页按钮                         | 保留搜索关键词和状态过滤，切换页码并请求后端 |
| 首次进入选课推荐标签页               | 搜索框为空，默认显示"待审核"状态的全部记录   |

## 七、后端 SQL 查询示例

以下为不同搜索场景下 `GetAll` 函数生成的 SQL 示例：

**场景 1：仅按状态过滤（当前行为，搜索框为空）**

```sql
SELECT COUNT(*) FROM course_recommendations WHERE is_visible = 0

SELECT ... FROM course_recommendations WHERE is_visible = 0
ORDER BY recommendation_time DESC LIMIT 6 OFFSET 0
```

**场景 2：按状态 + 课程名称搜索**

```sql
SELECT COUNT(*) FROM course_recommendations
WHERE is_visible = 0 AND course_name LIKE '%高数%'

SELECT ... FROM course_recommendations
WHERE is_visible = 0 AND course_name LIKE '%高数%'
ORDER BY recommendation_time DESC LIMIT 6 OFFSET 0
```

**场景 3：按状态 + 教师姓名搜索**

```sql
SELECT COUNT(*) FROM course_recommendations
WHERE is_visible = 1 AND teacher_name LIKE '%张%'

SELECT ... FROM course_recommendations
WHERE is_visible = 1 AND teacher_name LIKE '%张%'
ORDER BY recommendation_time DESC LIMIT 6 OFFSET 0
```

**场景 4：全部状态 + 同时搜索课程名称和教师姓名**

```sql
SELECT COUNT(*) FROM course_recommendations
WHERE course_name LIKE '%数学%' AND teacher_name LIKE '%李%'

SELECT ... FROM course_recommendations
WHERE course_name LIKE '%数学%' AND teacher_name LIKE '%李%'
ORDER BY recommendation_time DESC LIMIT 6 OFFSET 0
```

## 九、无需修改的部分

以下部分无需任何改动：

- **数据库表结构** - 现有 `course_recommendations` 表已包含 `course_name` 和 `teacher_name` 字段
- **model 层** (`model/course_recommendation.go`) - 无需新增请求/响应类型
- **路由** (`router/router.go`) - API 路径不变，仍为 `GET /api/v1/admin/course-recommendations`
- **中间件** - 认证逻辑不变
- **公告管理和系统配置** - 其他标签页不受影响
- **前端公共选课推荐页面** (`course-recommendation.html`) - 该页面有自己独立的搜索逻辑，不受影响
